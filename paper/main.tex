\section{Introduction}
\label{sec:intro}

Neural network quantization has emerged as a critical technique for deploying models on resource-constrained devices. While post-training quantization to 8-bit integers is well-established, recent work on extreme quantization---particularly 1-bit and 1.58-bit networks---has shown promising results for large language models~\cite{wang2023bitnet,ma2024bitnetb158}.

\textbf{Motivation.} The \bitnetb{} approach quantizes weights to ternary values $\{-1, 0, +1\}$, achieving a theoretical compression ratio of $20.3\times$ compared to FP32 weights. While this has been studied extensively for transformers, its application to convolutional neural networks remains underexplored.

\textbf{Gap.} Prior work on binary/ternary CNNs~\cite{courbariaux2015binaryconnect,rastegari2016xnornet} predates the \bitnet{} formulation and uses different quantization schemes. A systematic study of \bitnetb{} for CNNs---spanning multiple architectures, datasets, and training configurations---is missing from the literature.

\textbf{Contributions.} We make the following contributions:
\begin{enumerate}
    \item A systematic benchmark of \bitnetb{} applied to five CNN architectures across three image classification datasets.

\section{Related Work}
\label{sec:related}

\textbf{Binary and Ternary Neural Networks.}
BinaryConnect~\cite{courbariaux2015binaryconnect} introduced binary weights trained with the straight-through estimator (STE). XNOR-Net~\cite{rastegari2016xnornet} extended this to both binary weights and activations. Ternary Weight Networks~\cite{li2016ternary} showed that adding a zero value significantly improves accuracy over strict binary quantization.

\textbf{BitNet Family.}
BitNet~\cite{wang2023bitnet} introduced 1-bit weight quantization for large language models. \bitnetb{}~\cite{ma2024bitnetb158} refined this to ternary weights $\{-1, 0, +1\}$, achieving performance matching FP16 Transformers while reducing inference cost. Our work extends this to CNNs.

\textbf{Data Augmentation for CNNs.}
Standard augmentation (random crop, horizontal flip) is essential for CIFAR~\cite{krizhevsky2009cifar}. Advanced techniques like Cutout~\cite{devries2017cutout}, AutoAugment~\cite{cubuk2019autoaugment}, and RandAugment~\cite{cubuk2020randaugment} further improve generalization. We investigate how augmentation interacts with extreme quantization.

\section{Method}
\label{sec:method}

\subsection{BitNet Layer Implementation}

We implement \bitnetb{} for both linear and convolutional layers. The forward pass consists of three steps:

\textbf{1. Weight Quantization.} Weights are quantized to $\{-1, 0, +1\}$ using:
\begin{equation}
    \tilde{W} = \text{RoundClip}\left(\frac{W}{\gamma + \epsilon}, -1, 1\right), \quad \gamma = \frac{1}{n}\sum_{i}|W_i|
\end{equation}
where $\gamma$ is the mean absolute value and $\epsilon=10^{-5}$ ensures numerical stability.

\textbf{2. Activation Quantization.} Activations are quantized to 8-bit using absmax quantization:
\begin{equation}
    \tilde{x} = \text{Clip}\left(x \cdot \frac{Q_b}{\gamma_x}, -Q_b, Q_b\right), \quad Q_b = 2^{b-1}
\end{equation}
where $\gamma_x = \max(|x|)$ and $b=8$ bits.

\textbf{3. Dequantization.} After the matrix/convolution operation, outputs are scaled back:
\begin{equation}
    y = \frac{\gamma \cdot \gamma_x}{Q_b} \cdot (\tilde{W} \cdot \tilde{x})
\end{equation}

\textbf{Gradient Flow.} The straight-through estimator (STE) allows gradients to flow through quantization by using the identity function in the backward pass.

\subsection{Model Conversion}

We convert standard architectures from the \texttt{timm} library~\cite{rw2019timm} by replacing all \texttt{nn.Linear} and \texttt{nn.Conv2d} layers with their BitNet equivalents. Batch normalization and activation functions remain in FP32.

\subsection{Training Protocol}

All models are trained from scratch with:
\begin{itemize}
    \item Optimizer: SGD with momentum 0.9 and Nesterov acceleration
    \item Learning rate: 0.1 with cosine annealing
    \item Weight decay: $5 \times 10^{-4}$
    \item Epochs: 200 (CIFAR), 90 (ImageNet)
    \item Batch size: 128
\end{itemize}

\subsection{Experimental Setup}

\textbf{Datasets.} We evaluate on CIFAR-10 (10 classes, 50K train), CIFAR-100 (100 classes, 50K train), and ImageNet-1k (1000 classes, 1.28M train).

\textbf{Models.} We compare five architectures: ResNet-18, ResNet-50~\cite{he2016resnet}, VGG-16~\cite{simonyan2014vgg}, MobileNetV2~\cite{sandler2018mobilenetv2}, and EfficientNet-B0~\cite{tan2019efficientnet}.

\textbf{Augmentation Levels.} We ablate four augmentation configurations:
\begin{itemize}
    \item \textbf{Basic}: RandomCrop(32, padding=4) + RandomHorizontalFlip
    \item \textbf{RandAug}: Basic + RandAugment(n=2, m=9)
    \item \textbf{Cutout}: Basic + RandomErasing(p=0.5)
    \item \textbf{Full}: RandAugment + RandomErasing
\end{itemize}

\textbf{Seeds.} All experiments run with seeds $\{42, 123, 456\}$. We report mean $\pm$ std.
